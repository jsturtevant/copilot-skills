## Inconsistencies & Centralization Opportunities

### Duplicated Code
The two backend scripts share **~80% identical code** (~679/853 lines). Duplicated functions include: `discover_tools()`, `discover_mcp_servers()`, `_check_workspace()`, `_glob()`, `_web_fetch()`, `_html_to_text()`, `_bash()`, `_sql()`, `_github_api()`, `apply_user_config()`, `_load_user_tool()`, `_user_config_dir()`, `_split_csv()`, `_make_handler()`.

**Recommendation:** Extract into `scripts/codeact_common.py`. Each backend becomes a thin ~150-line wrapper for sandbox-specific registration.

### API Inconsistencies
| Feature | Monty | Hyperlight |
|---------|-------|------------|
| web_fetch `max_length` param | ✅ In manifest | ❌ Missing from manifest |
| web_fetch description | ✅ Mentions HTML stripping | ❌ Generic "fetch URL" |
| Type annotations | ❌ None | ✅ Full Python 3.10+ |
| Brace expansion docs | ❌ Says it fails (wrong) | ✅ Says it works |
| Create error message | ⚠️ No guidance | ✅ Suggests using edit |

### Error Handling
- MCP config parsing: `except Exception: pass` — completely silent
- Custom tool loading: warns but continues — acceptable
- Tool execution: inconsistent between backends

### Build / CI Gaps
- No GitHub Actions workflows
- No linter (ruff, pylint, mypy)
- No pre-commit hooks
- Unit tests cover only tool config (14 tests), not core logic
- No automated backend parity checks

## Recommendations (Priority Order)

1. **Fix path traversal** — Use `Path.is_relative_to()` or append `/` to workspace root (Critical, low effort)
2. **Restrict MCP bridge env vars** — Allowlist safe vars, don't pass full `os.environ` (Critical, low effort)
3. **Add URL scheme validation** — Only allow http/https in `_web_fetch` and MCP bridge (Critical, low effort)
4. **Add file size limits to _view()** — Cap at 50MB or configurable limit (Critical, low effort)
5. **Cap MCP bridge response reads** — `resp.read(10*1024*1024)` for 10MB max (Critical, low effort)
6. **Extract shared code into codeact_common.py** — Eliminate 80% duplication (Critical maintainability, medium effort)
7. **Fix brace expansion docs in monty SKILL.md** — Remove false "fails" claim (Low, trivial)
8. **Add basic CI workflow** — Unit tests + lint on PR (Medium, low effort)
9. **Add --max-time and --max-redirs to curl** — Harden web_fetch (High, low effort)
10. **Clear _SQLITE_CONNECTIONS between runs** — Prevent data leakage (Critical, low effort)
